## 补充文档：终端原生体验、自诊断修复与扩展集成（v1.0）

本文档是对《超级万能AI Agent 产品设计文档（v2.3）》及其心跳/Cron、社交网关补丁的进一步扩充，旨在补齐 **终端交互界面(TUI/CLI)**、**自诊断与自修复**、以及**操作系统级与开发者生态集成**等关键体验和能力，确保 Agent 在工程品质和全场景覆盖上达到行业顶级水准。

> **设计原则**：所有新增功能均遵循“单二进制、零外部依赖”原则，与现有系统无缝融合，不引入额外运行时。终端体验复用 Go 生态成熟库，诊断能力内置于核心二进制中。


### 1. 终端原生体验 (TUI/CLI)

#### 1.1 目标

- 提供**无需浏览器**即可与 Agent 交互的终端界面（TUI），适用于服务器、SSH 环境或极客用户。
- 提供一套完备的**命令行工具集 (CLI)**，用于管理 Agent 生命周期、监控状态、执行运维操作，类似 `openclaw` 子命令风格。

#### 1.2 TUI（终端用户界面）

**核心能力**：
- 终端内交互式对话窗，支持多会话切换，实时流式输出。
- 终端内查看 Agent 列表、团队状态、心跳与 Cron 记录。
- 与 Web 版数据完全同步（共享同一后端与数据存储）。

**技术选型**：Go 生态的 `bubbletea` (TUI 框架) + `lipgloss` (样式) + `bubbles` (组件)。  
**所需依赖**：`github.com/charmbracelet/bubbletea`、`github.com/charmbracelet/lipgloss`、`github.com/charmbracelet/bubbles`。

**功能点**：
- **启动命令**：`agent tui` 或 `agent` 无参数时自动检测终端，若无图形环境则进入 TUI 模式。
- **主面板**：左侧会话列表（支持搜索），右侧对话区域，支持 Markdown 渲染（终端兼容）。
- **快捷键**：`Ctrl+N` 新建会话，`Ctrl+W` 关闭会话，`Ctrl+C` 退出，`Ctrl+S` 保存对话历史。
- **系统菜单**：`F1` 查看 Agent 状态，`F2` 查看定时任务，`F3` 查看心跳日志。

**通过 `//go:embed` 嵌入静态资源？** 无需，TUI 纯由 Go 端渲染。

#### 1.3 CLI（命令行工具集）

**设计哲学**：所有子命令遵循 `agent <command> [options]` 格式，支持 `--json` 输出以利脚本集成。

**命令清单**（对标 OpenClaw CLI）：

| 命令 | 功能 | 主要参数 |
|------|------|----------|
| `agent start` | 启动 Agent 运行时（Web + 后台服务） | `--port`, `--daemon` |
| `agent stop` | 停止正在运行的 Agent 实例 | |
| `agent restart` | 优雅重启 Agent（保持会话不中断） | |
| `agent status` | 显示运行状态：运行时间、内存、活跃会话、最新心跳 | `--json` |
| `agent list` | 列出所有 Agent 实例及其角色、状态 | `--team <name>` |
| `agent doctor` | 运行自诊断（详见第 2 节） | `--fix` 自动修复 |
| `agent logs` | 查看 Agent 运行日志 | `--follow`, `--agent <id>` |
| `agent config` | 查看或修改配置项 | `--get <key>`, `--set <key=value>` |
| `agent cron` | 管理定时任务 | `list`, `add`, `remove`, `run` |
| `agent gateway` | 管理社交网关通道 | `status`, `enable`, `disable`, `restart` |
| `agent workspace` | 管理工作空间 | `export`, `import`, `prune` |
| `agent update` | 检查并更新 Agent 自身二进制 | `--channel stable/beta` |

**示例**：
```bash
agent start --port 8443 --daemon
agent doctor --fix
agent cron add "0 9 * * *" "提醒我看报告" --target telegram
```

**技术实现**：基于 Go 标准库 `flag` 或使用 `cobra` 框架（可根据复杂度权衡，但为了零依赖可先用标准库，若子命令过多再引入 cobra）。TUI 与 CLI 共享同一可执行文件，通过子命令区分。


### 2. 自诊断与自修复 (Self-Healing)

#### 2.1 目标

- 提供 `agent doctor` 命令，主动扫描系统健康状态，识别配置错误、依赖缺失、权限问题等常见问题并给出修复建议。
- 提供运行时状态监控 (`agent status`)，实时展示资源使用与关键指标。
- 优雅重启机制，保证更新或配置变更时不丢失进行中的会话数据。

#### 2.2 `agent doctor` 诊断项

诊断引擎以插件式模块组织，覆盖以下检查（对标 OpenClaw `doctor` 命令）：

| 诊断项 | 检查内容 | 严重级别 | 自动修复（`--fix`） |
|--------|----------|----------|---------------------|
| `config` | 检查 `agent.yaml` 格式有效性、必填项 | 错误 | 尝试恢复默认配置 |
| `api_keys` | 验证已配置的各 LLM 提供商 API Key 是否有效（发起简短测试请求） | 错误 | 无法自动修复，提示重新设置 |
| `workspace` | 检查工作空间目录是否存在、可读写 | 错误 | 创建缺失目录并修正权限 |
| `db` | 检查 SQLite 数据库完整性，尝试修复 | 警告 | 执行 `PRAGMA integrity_check` 并尝试修复 |
| `sandbox` | 验证当前平台的原生沙箱机制是否可用 | 警告 | 降级为 `off` 模式并通知用户 |
| `gateway` | 测试社交网关各通道连接状态（如 Telegram Webhook） | 警告 | 尝试重新注册 Webhook |
| `cron` | 检查 Cron 任务存储完整性，是否有错过调度 | 信息 | 重新计算并补发未执行任务 |
| `memory` | 统计工作空间记忆文件大小、向量库状态 | 信息 | 提示清理无用的记忆碎片 |
| `connectivity` | 检查核心 LLM API 端点可达性 | 错误 | 提示网络问题，无法自动修复 |
| `disk_space` | 检查数据目录所在磁盘剩余空间 | 警告（<500MB） | 建议用户清理或迁移 |
| `version` | 检查当前版本是否为最新稳定版 | 信息 | 提供 `agent update` 命令 |

**输出格式**：终端表格展示，每项显示状态图标（✅/⚠️/❌）、诊断信息、修复建议。`--json` 可输出结构化 JSON。

**实时运行中诊断**：除了手动执行，Agent 运行时每隔 1 小时自动静默执行部分轻量级诊断（如 `disk_space`、`connectivity`），发现问题通过心跳或通知推送用户。

#### 2.4 优雅重启

- **graceful restart**：收到重启信号后，停止接收新请求，等待所有进行中的 Agent 循环、心跳、Cron 任务完成（最长 60s 超时），然后退出旧进程。新进程接管端口，加载持久化的会话状态，继续服务。
- **实现方式**：使用 `os.Signal` 捕获 `SIGTERM`/`SIGINT`；利用 `sync.WaitGroup` 等待活跃任务；支持通过 Unix Domain Socket 传递监听 fd（可选，用于零停机重启，但复杂度高，暂定为常规优雅关闭+立即重启）。


### 3. 扩展操作系统级集成

#### 3.1 IDE 扩展与协议桥接

**目标**：让 Agent 能直接参与开发者的编码流程，与 VSCode、Cursor、Windsurf 等 AI 编程工具打通，作为项目级的后台智能体。

**实现方式**：
- **MCP 服务器暴露**：Agent 内嵌 MCP 服务器端，标准输入/输出模式，可被 Cursor、Windsurf 等直接添加为 MCP Server。通过 MCP 提供以下能力：
  - `get_project_context`：根据当前 Git 仓库返回 AGENTS.md 规则、项目结构、最近提交等上下文。
  - `run_test`、`lint`、`typecheck`：触发 Agent 的工具调用执行验证管道，并将结果返回 IDE。
  - `delegate_task`：将 IDE 中选中的代码或自然语言任务委托给 Agent 团队，执行后返回结果。
- **A2A 协议扩展**：支持 IDE 作为 A2A 客户端，通过 A2A 协议向 Agent 提交任务（如“审查我修改的这段代码”）。
- **配置方式**：在 IDE 的 MCP 配置文件中添加：
  ```json
  "mcpServers": {
    "agent": {
      "command": "agent",
      "args": ["mcp-server"]
    }
  }
  ```
  之后 IDE 即可发现 Agent 提供的能力。

#### 3.2 桌面应用与系统守护进程

虽然强调单二进制，但可提供**可选的桌面体验增强**：
- **macOS Menu Bar App**：一个极简的托盘图标，展示 Agent 状态，提供快捷打开 Web 面板、退出等菜单项。通过 Go 编译为独立二进制（可改为同一二进制通过子命令启动托盘应用）。
- **系统级守护进程**：通过 `agent start --daemon` 自动注册为 systemd（Linux）、launchd（macOS）、Windows Service，遵循操作系统规范，无需额外工具。

#### 3.3 多模态内容生成（扩展工具）

**目标**：允许 Agent 调用云端 API 生成图片、视频、音乐（对标 OpenClaw 的原生 `image_generate` 等工具）。

**工具注册**：
- `image_generate`：调用 DALL·E、Stable Diffusion API 等生成图片，支持尺寸、风格等参数。
- `video_generate`（可选）：对接 Runway、Pika 等 API。
- `music_generate`（可选）：对接 Suno、Udio 等 API。

这些工具作为可选技能包提供，不作为核心二进制的一部分，但工具定义和配置内置于默认工具市场中，用户可一键安装对应技能包。

#### 3.4 子智能体编排工具增强

**目标**：将多 Agent 协作能力更深入地暴露给 Agent 自身，让其能在对话中动态创建子 Agent、分配任务、检查进度（类似 OpenClaw 的 `subagents` 工具）。

**新增内置工具**：
- `spawn_subagent`：基于自然语言角色描述和可选工具集，动态创建一个临时子 Agent，返回其 ID。
- `delegate_to_subagent`：将任务委托给指定子 Agent，返回执行结果。
- `list_subagents`：列出当前存在的子 Agent 及其状态。
- `terminate_subagent`：销毁一个子 Agent，释放资源。

**安全边界**：子 Agent 继承父 Agent 的权限子集，默认沙箱模式为 `native`，工作空间为临时目录，完成后自动清理。


### 4. 实施路线图

| 阶段 | 内容 | 优先级 | 预估工期 |
|------|------|--------|----------|
| Phase 1 | **CLI 基础框架**：`start`, `status`, `list`, `logs`, `doctor`, `config` 命令 | P0 | 3天 |
| Phase 2 | **TUI 对话界面**：基于 bubbletea 构建基本聊天和状态面板 | P1 | 5天 |
| Phase 3 | **自诊断增强**：实现全部诊断项，`--fix` 自动修复 | P0 | 2天 |
| Phase 4 | **优雅重启**：完善信号处理和会话保存逻辑 | P0 | 2天 |
| Phase 5 | **IDE MCP 服务器**：实现 MCP 端点和项目工具 | P1 | 3天 |
| Phase 6 | **子智能体编排工具**：实现动态创建/销毁子 Agent | P1 | 4天 |
| Phase 7 | **多模态生成工具包**（作为技能包发布） | P2 | 按需 |
| Phase 8 | **桌面应用托盘**（可选，macOS 先行） | P2 | 3天 |

> 以上所有功能均保持单二进制交付，依赖库均为 Go 生态，无外部运行时需求。所有命令和交互均与 Web 版共享同一后端 API，保证功能一致。
